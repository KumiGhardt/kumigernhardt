Mybrary Case Study

Mybrary is a React web app developed using the MERN stack, that provides users with access to information about movies, directors, and genres. Users are able to create an account, update their personal data, and create a list of favorite movies. Mybrary was a personal project I built as part of my web development course at CareerFoundry as a demonstration of my understanding of the React framework.

The aim of the project was to build a client-side application based on its existing server-side code (REST API and database). I had previously created a RESTful API using Node.js and Express, that interacts with a non-relational database (MongoDB). The API can be accessed via commonly used HTTP methods and CRUD methods are used to get access to the data. I started to build the interface users would need when making requests to, and receiving responses from, the server-side. It is a single-page, responsive application, developed with React and React-Redux.

The technical requirements for this project were:

  • The application must be a single-page application (SPA)
  • The application must use state routing to navigate between views and share URLs
  • The application must give users the option to filter movies
  • The application must give users the option to sort movies
  • The application must initially use Parcel as its build tool
  • The application must be written using the React library and in ES2015+
  • The application must be written with React Redux (hence respecting the Flux pattern)
  • The application must use Bootstrap as a UI library for styling and responsiveness
  • The application must contain a mix of class components and function components
  • The application may be hosted online

This was my most challenging project. React challenged my knowledge of the concepts of Javascript and I was intrigued by the idea of building complex UIs from components. It however was quite challenging for a time for me to get. In fact the first attempt at mybrary I imagine looks like a badly constructed lego car next to the porsche equivalent of what can be built by React. I solved these challenges by learning the importance of reading the documentation, learning how to use console.log and learning when to ask for help from more experienced developers and my peers. It took me a full month to fully complete the project but the interesting thing is once I learned how to learn this, doing the same project in Angular was so much easier!
